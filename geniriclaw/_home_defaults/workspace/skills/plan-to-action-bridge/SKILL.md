# plan-to-action-bridge — мост от плана к исполнению

> **Status:** stable
> **Category:** meta · execution-discipline

## Purpose

Предотвращение типовой ошибки агента — «написать план, отчитаться «план готов» и остановиться». Каждый planning-шаг **обязан** заканчиваться `NEXT_EXECUTABLE_STEP` с доказательством старта реальной работы (pid, session_id, file_path, url).

## Fail modes it addresses

1. **«План без действия»** — агент пишет «я подготовил план из 5 шагов», но ни один шаг не запущен.
2. **«Отчёт вместо артефакта»** — агент пишет «проанализировал 3 варианта» без конкретных файлов/ссылок.
3. **«Промежуточный ОК»** — агент спрашивает «всё понятно? можно запускать?» вместо того чтобы запустить и показать результат.
4. **«Работа в голове»** — агент «спроектировал» что-то, но ни одного файла/скрипта/коммита не появилось.

## Bridge rule

**После любого planning-like шага** в том же ответе обязателен блок:

```markdown
## NEXT_EXECUTABLE_STEP

<one-line summary of what is being executed right now>

Evidence:
- pid: 12345 (background task started)
- session: main-3ae44aee
- file: /path/to/result.txt (written at 12:34:56)
- url: https://{{YOUR_DOMAIN}}/docs/xxx.html (published)
- log: /tmp/task-xxx.log (tail -f для наблюдения)
```

Минимум **одно** из evidence-полей должно быть заполнено.

## Allowed evidence (что считается доказательством)

| Тип | Пример | Что доказывает |
|-----|--------|----------------|
| `pid` | `pid: 12345` | Subprocess реально запущен |
| `session_id` | `session: foo-bar-baz-42` | Claude CLI session активна |
| `file_path` | `/absolute/path/file.ext` (с `ls -l` / `stat` подтверждением) | Файл создан |
| `url` | `https://example.com/...` (с `curl -sI` 200) | Артефакт доступен публично |
| `log_tail` | `tail -f /tmp/task.log` показывает новые строки | Процесс живой |
| `message_id` | TG `message_id`: 42 | Сообщение отправлено |
| `commit_sha` | `git log -1 --oneline` | Коммит сделан |

## НЕ-evidence (provide but keep working)

Эти вещи упоминать МОЖНО, но они **не** заменяют evidence:

- «Я подумал...»
- «План такой: ...»
- «Планирую сделать X»
- «Всё готово для запуска»
- «Ждём OK?»

## Minimal handoff pattern

```
[agent plans something]
↓
[agent provides plan in structured form]
↓
NEXT_EXECUTABLE_STEP: <start of real execution in same response>
↓
[evidence collected]
↓
[agent reports status with real artifacts]
```

## Для storyboard → production перехода

Частый провал: раскадровка одобрена → агент «готов переходить к генерации» → ждёт OK → ничего не делает.

**Правильно:** в том же ответе где пришло одобрение — сразу запуск batch generation с выводом `pid` + `session` + ссылкой на progress-файл. Оператор видит что работа пошла и может следить.

## Обязательная самопроверка

Перед отправкой ответа, агент проверяет:

- [ ] Есть ли в ответе реальное действие (tool call с последствиями), не только текст?
- [ ] Если есть секция «план» / «предлагаю» / «могу сделать» — есть ли после неё evidence что уже **делается**?
- [ ] Если результат «готов» — есть ли конкретная ссылка/путь/id, который оператор может открыть?

Если хотя бы на одно «нет» — доделать перед отправкой.

## Repair protocol after gap

Если агент поймал себя на том что отчитался без действия:

1. Не извиняться длинно — одной фразой признать gap.
2. Немедленно в том же ответе запустить то что обещал.
3. Собрать evidence.
4. Пойти дальше — не зацикливаться на саморефлексии.

## Rules

- **Промежуточный ОК запрошен только** при необратимых внешних действиях: публикация в соцсети, DM реальным людям от лица оператора, траты > $10, деструктивные операции.
- **Для всего остального** — делаем молча, показываем готовый результат.
- Планирование в длинных задачах — ок, но каждая phase заканчивается NEXT_EXECUTABLE_STEP.
- Parallel execution: если есть 3 независимых шага — запустить все 3, потом дождаться и собрать.

## Anti-patterns (запрещено)

| Фраза | Почему запрещено | Чем заменить |
|-------|------------------|--------------|
| «сейчас сделаю» | Обещание вместо действия | Сделал + evidence |
| «проверю» | Обещание | `[Tool: Bash] → результат` |
| «подготовила» | Процесс без артефакта | Путь к файлу / message_id |
| «почти готово» | Пустой отчёт | `X% shots done, next: shot_Y at pid Z` |
| «могу запустить?» | Лишний confirm | Запустить + показать результат |
