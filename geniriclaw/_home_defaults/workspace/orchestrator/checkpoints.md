# checkpoints.md — что отправлять оператору на каждом checkpoint

Этот файл описывает **формат** сообщения оператору когда pipeline доходит до `checkpoint: control` — чтобы не пересылать весь state.yaml, а показать именно то что нужно для решения.

## Формат control-сообщения

```
<stage-icon> <stage-name> готово
<артефакт-ссылка / preview>

<1-2 строчки ключевого решения которое надо принять>

Ответь:
  • <вариант A (обычно: go)>
  • <вариант B (обычно: правки в свободной форме)>
```

## Checkpoints по стадиям

### script (после scriptwriter)

```
✏️  Script готов — 45 сек, 10 shots
📄 drafts/<job>/script.md

Ключевые моменты: hook про <X>, развитие через <Y>, финал на <Z>.
Длительность укладывается в 45с (средний темп 2.8 слова/сек).

Go — переходим к sketches (10 мин)
Правки — пришли цитатой что поменять
```

### sketches (после image-gen sketches batch)

```
🎨 Sketches готовы — 10 кадров, B&W
🔗 https://<your-domain>/sketches/<job>/

Проверь композицию и сюжет. Лица не важны на этом этапе.

Go — финальный рендер (25 мин, $0.12)
Правки — укажи shot_NN и что поменять
```

### edit (финальный concat)

```
🎬 Финал готов — 45с, 9:16
🎥 drafts/<job>/build/final.mp4 (6.2 MB)
🔗 превью: https://<your-domain>/published/<job>/preview.mp4

Audio mix: voice -1dB, music -18dB (ducking).
Director's cut transitions: <сколько> переходов добавлено.

Go — публикация в TG/IG/TikTok
Правки — например "shot_03 слишком долго, обрежь 1с"
```

### publish (перед отправкой в публичные каналы)

```
📤 Готов к публикации

Caption (preview):
---
<текст подписи, первые 200 символов>
---

Платформы: TG channel @xxx, IG @yyy, TikTok @zzz

Go — публиковать во все 3
Только TG / только IG — выбрать какие
Stop — отложить, не публиковать
```

## Правила формирования checkpoint-сообщений

1. **1 preview артефакт** — не перечислять все файлы.
2. **Короткая суть решения** — не пересказывать весь stage.
3. **Явные варианты ответа** — `go` / правки / `stop`.
4. **Метрики если релевантны** — длительность, размер, стоимость.
5. **Нет технических деталей** без запроса — pid, session_id, внутренние пути в чат не идут.

## Сообщения-статусы в autopilot

Чтобы оператор видел прогресс без спама — только **ключевые точки**:

- **Начало job**: «Взяла бриф: <тема>. Mode: Autopilot. Первое промежуточное — sketches, ~10 мин.»
- **Через каждые ~10 мин**: «Промежуточно: shots 5/10 готовы»
- **На control-точках**: см. форматы выше
- **Ошибки, которые применили fallback**: «shot_03 переделала с упрощённым промптом — прошло. Остальные ок.»
- **Ошибки, требующие оператора**: «shot_05 не проходит safety 3 раза подряд. Варианты: (a) удалить из ролика, (b) перепи́сать сцену полностью, (c) заменить на still.»

## Сообщения-НЕсообщения

**Не писать оператору** в чат:

- Стандартные промежуточные логи (idle polling, cache refresh)
- Каждый успешный HTTP-запрос к провайдеру
- Нормализация аудио / resize картинок
- Полный state.yaml дамп

Это всё пишется в `agent.log`, в чат — нет.
